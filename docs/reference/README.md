# Reference

Use this directory for stable, descriptive project knowledge such as:

- terminology and glossary entries
- environment variable contracts
- service and integration inventories
- API and event contracts
- data ownership and schema maps
- reusable repository process prompts and contracts
- authoritative external source links with review dates

Reference documents describe facts and interfaces. Put architectural rationale
in `docs/decisions/`, current state in `docs/context/`, and procedures in
`docs/runbooks/`.
